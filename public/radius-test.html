Corner radius calibration

Option A tokens — one var per size, swapped via @supports so browsers without corner-shape get a smaller plain rounded corner that looks visually equivalent to the inflated squircle version used by supporting browsers.

corner-shape: superellipse(1.6)

Side-by-side at each size

Column 1 — plain rounded at the fallback px. Column 2 — squircle 1.6 at the same (fallback) px: shows the visual mismatch when you don't inflate. Column 3 — squircle 1.6 at the calibrated (inflated) px: the size that should look like column 1. Column 1 and Column 3 should look identical in size; adjust the calibration px values if they don't.
